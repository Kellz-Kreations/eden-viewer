First run setup

Fill in values and click Download .env. Then place the downloaded .env next to compose.yaml and redeploy.

What this does
  • Generates a .env file for this repo and downloads it to your computer.
  • Prefills defaults from .env.example if the repo is mounted.
What this doesn’t do
  • Does not save your inputs on the server (including PLEX_CLAIM).
  • Does not configure Sonarr/Radarr/Plex beyond writing .env.
  • Intended for LAN use (don’t expose this UI directly to the internet).
Next steps
  1. Download .env and place it next to compose.yaml.
  2. If you started the setup-only project: run docker compose -f compose.setup.yaml down.
  3. Start the full stack: docker compose up -d (or deploy via DSM Container Manager).
